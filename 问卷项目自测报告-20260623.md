# 问卷项目自测报告

## 严重问题（必须修复，影响正常使用）

| 编号 | 问题描述 | 复现步骤 | 影响范围 |
|------|--------|--------|--------|
| 无 | 未发现阻断正常创建、发布、填写、提交、查看记录的严重问题 | 已覆盖正常填写、必填校验、显示条件、极端输入、移动端填写和提交记录查看 | 无 |

## 一般问题（建议修复，影响使用体验）

| 编号 | 问题描述 | 复现步骤 | 影响范围 |
|------|--------|--------|--------|
| P2-1 | 填写页未提交内容在浏览器离开后返回会丢失 | 打开填写页，输入文本并选择选项，跳转到其他页面后点浏览器后退，已填内容清空 | 所有填写页用户，尤其是手机端误触返回/切换页面场景 |
| P2-2 | 填写页提交按钮没有提交中禁用态，快速重复点击可能发出重复请求 | 填写后快速多次点击提交；后端会用唯一提交限制挡住重复提交，但前端没有 loading/disabled 反馈 | 粗心用户/网络慢时体验可能困惑 |

## 优化建议（锦上添花，可选做）

| 编号 | 建议内容 | 预期效果 |
|------|--------|--------|
| S1 | 为填写页增加本地草稿缓存，按 `shareToken` 暂存未提交答案 | 解决后退、刷新、误关闭导致的答案丢失 |
| S2 | 提交按钮增加 loading 和 disabled 状态 | 避免重复点击，反馈更明确 |
| S3 | 修复/更新后端 e2e 测试配置；当前 `npm run test:e2e` 因缺少 `DATABASE_URL` 无法运行 | 提高后续回归测试可信度 |
| S4 | 明确本地/部署环境的 `FRONTEND_ORIGIN` 配置，避免 5173/5174/8080 端口切换导致 CORS 风险 | 减少换环境后的接口访问问题 |
